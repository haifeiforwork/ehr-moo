1. Message Overview

Message function is to support efficient communication between members. Notice has various uses from simple message to notice distribution within monthly message capacity (10GB). Message can be divided by writer according to the importance of contents, and received message can be checked in information window in real time. In addition, message considered to be importance by a user can be separately stored in Archive Box (1GB). The administrator can set monthly message capacity and Archive Box capacity.

Configuration of Message main screen

The figure below is configuration of Message main screen. It shows list in the right side besides left menu area. Click the list to display message detail as shown in the figure below.

① unRead : Display only unread message.
② search : This function is to search message desired to be checked. You can search by message category (general, notice, information, and distribution), sender, and contents.
③frame change : This is to change screen structure (up/down/right/left). Currently saved message list is displayed on the 'top/left’ of screen. On the message contents will be displayed on the 'bottom/right'.
④ message list : This is to display message list in the relevant message box. Urgent, read/unRead, category, sender, contents, date, and attachment capacity are displayed. You can set number of messages per page and delete or store the selected message.
⑤ message contents : Display the contents of a message selected by the user.

2. Message menu configuration

Message menu consists of menu provided to all users and management menu provided to administrator. Administration menu group for administrator will be provided only for the user with administration authority.

① message composition : User can write and send a new message.
② Inbox( received message list ) : Display received message list. It provides numbers of displayed message, urgent/unRead check, or searched message list according to setting condition. Received message overdue period set by administrator will be automatically deleted from Inbox. If a user wants to keep message regardless of period, the user has to move it into Archive Box.
③Outbox( sent message list ) : Display message list sent by a user. You can check for receiving status of sent message and (numbers of displayed message, urgent/unRead check), or searched message list will be provided according to setting condition. Sent message overdue period set by administrator will be automatically deleted from Outbox. If a user wants to keep message regardless of period, the user has to move it into Archive Box.
④ Archive Box( archived message list ) : This is to display archived message list among sent or received messages. Message stored in Archive Box won’t be deleted until it is deleted by the user.

3. Composing / sending

Click ‘compose’ on the left menu to display pop-up window in which you can compose and send a message. A user can enter contents and attach file to send a message.

The followings are description of each input item based on example screen for Message composition.

① Urgency : In case of urgent message, check at ‘Urgent’ for the sent message. '!’ mark will be displayed on receiver’s screen.
② Division : You can select one of 'general/notice/information/distribution' according to message. 'General' is set as default.
③receiver selection : Select receiver (s) for the message to be sent. To add receiver(s), click 'Address' icon to call organization chart screen and then select user(s) on organization chart or search for user(s) (Max 500 users). To cancel selected user, select desired user (s) in ‘Select’ box and then click 'Delete' button.
④ contents input : Enter message contents.
⑤ file attachment : You can select and attach document.

4. Receipt notification

Receipt of message will be displayed on the right bottom of a browser in information window in real time. You can click information window to check message contents and can reply received message.

5. Making use of message function

Each box type for received message has various functions. A user can manage message using functions such as mail, reply, forward, store, delete, and retrieve.

Sending reply

Forwarding

This function is to forward message to another person.

Archiving

This function is to move message in Inbox into Archive Box. Since received message will be automatically deleted according to the term set by the administrator, important message must be stored in Archive Box.

Retrieving

This function is to retrieve sent message. message read by receiver won’t be retrieved, and read/unread status can be checked through '수신확인' of the relevant message.
