1. Workspace outline

Workspace is a space for collaboration activity of organization unit. Like team building of a company, it supports activity space of virtual organization such as team unit knowledge activity and communication invigoration space, TFT that acts with a purpose for a specific period and informal gathering activity space such as in-house club.

Workspace screen example

① menu area : consists of function of notice, board configuration, activity statistics
② Intro area : consists of simple introduction or image of workspace
③ main area : main screen of workspace arranged with many potlets

2. opening and closing of Workspace

2.1 Workspace opening

When a team is organized, Team Workspace will be automatically opened. In case of TFT, Café besides Team Workspace will be finally opened after obtaining approval of manager at the time of workspace opening request.

Workspace opening process

① Workspace opening request :
- click ‘Opening request of workspace’ button in total workspace main screen
- registrant will be automatically designated as sysop after completing opening
② Member recruit and collaboration environment construction :
- sysop will compose collaboration environment such as member management, main screen configuration, and board configuration.

2.2 Workspace closing

Only sysop can request for Workspace closing. Workspace will be finally closed after data migration and manager’s approval. Access to workspace is possible even after closing request. However, access is not possible after final closing and all data will be discarded.

Workspace closing process

① Workspace closing request :
- Only sysop is allowed to request for Workspace closing.
- In case data migration is required, request for closing after completing migration to other workspace
- workspace access and data migration is possible even after closing request.
② closing completed :
- workspace will be finally closed by manager’s closing approval
- When closing is completed, all boards and data will be closed, too.

3. Member joining and withdrawal

In case of Team workspace, a user will be automatically joined in a workspace of his/her own team as a regular member. In case of TFT or Café, a user has to apply for member and obtain administrator’s approval.

4. Workspace management

4.1 member management

A sysop and an administrator has member management authority for joining of workspace member and adjusting of member level, etc.

Workspace member management

① mail sending : send group mail to Workspace members
② New member registration : sysop/administrator is allowed for member batch registration
③ member level adjustment : change member level such as sysop/administrator/regular member/associate member
④ Withdrawal by force : forced withdrawal of specific member

4.2 board management

An sysop and administrator can form board tree according to characteristic of workspace for smooth collaboration in workspace.

Workspace board management

① How to add board :
- Right-click on folder in board tree display add menu.
- board creation: menu selected to create board for article registration
- category creation: menu to create Dummy folder for article registration
- link creation: menu selected to create folder that is linked to a specific URL
② How to search/modify/delete board :
- click folder in the board tree for detailed information search and to modify/delete it.
- Right-click on folder in the board tree to display modify/delete menu.
③ How to move board :
- Select board folder desired to be moved and Drag & Drop it.

4.3 main screen management

sysop and administrator can compose main screen by arranging proper potlets. all boards in workspace can be added using potlet.

Workspace main screen management

① introduction screen edit :
- You can arrange image and text on the upper part of workspace main screen.
② layout management :
- Drag & Drop default potlet and board potlet to arrange screen.

5. Communication between upper and sub team

5.1 notice distribution

Team unit workspace is an independent work space, but it also has hierarchical relationship between upper and sub organization(team). Therefore, we can easily perform notice distribution down to sub organization.(Team workspace Only)

notice

① sub organization sharing : ‘Share with sub organization’ button will be displayed when you search notice in workspace of a team that has sub organization.
② Click ‘share with sub organization’ button to select workspace for notice distribution in all sub organization list.

5.2 weekly report

This screen has function to register and collect weekly report is provided in team workspace. Only an sysop or administrator can collect weekly report, and collected weekly report will be opened in workspace of upper organization. (Excluding TFT, Café)

collecting and sharing of weekly report

① weekly report writing : Use ‘Register weekly report’ button next to your name
② weekly report collection : Use ‘Collect’ button in weekly report list
③ collected weekly report will be searched in upper organization workspace.

6. workspace alliance

6.1 alliance process

Alliance means to make a relationship with other workspace for mutual collaboration and data sharing. It is a good way to share data easily without risk of security.

alliance process

① You can search articles in a specific board that are opened in the relevant workspace through forming alliance with other workspace.
② You can set board opening for alliance workspace in board management screen.

6.2 data sharing between alliance

Each alliance workspace can give board search permission to another alliance workspace so that they can search data in another workspace. However, an sysop and administrator can provide board search permission to alliance.

data sharing between alliance

① After forming alliance, sysop/administrator will set opening of data in the relevant board of alliance workspace during creating or modifying board.
② list of board shared in My workspace menu area will be displayed when board is opened by alliance workspace.
