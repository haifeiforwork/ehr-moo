1. Q&A Overview

Q&A supports activity for problem-solving of organization members by asking necessary detail and answering them. For quicker and more efficient sharing and getting answer, you can spread question and answer detail via many media such as message, SMS, mail, and/or Microblog. In addition, in case there is other previous similar question or related answer by other users, they will be also provided to all users to solve similar or repetitive questions and issues all together.

Q&A Steps

Once a question is registered in Q&A, many users will provide answers for problem-solving. In case there is no first answer yet, the relevant question will be in 'Not answered' status. The questioner can ‘Adopt’ the most helpful answer among the provided answers. Now, the relevant question will be considered as ‘Completed’. In case the questioner has not adopted any answer among answers to the question, the relevant question will be still in 'Unsolved' status.

2. Q&A Menu

Q&A menu consists of menu provided to all users and management menu provided only for administrator. Administration menu group that is provided only for administrator will be provided only to user with administration authority.

① My question & answer(Question and answer list of you to see your Q&A activities) : This is to display numbers of questions registered by you and answers provided by you. Click it to move to the screen for your question list and question list to which you provided answer.
② Urgent question(List of questions that are checked as urgent question) : List of questions that are checked as urgent questions by questioner at the time of question registration. Urgent questions will be displayed in the order of recently registered question.
③ Unsolved question(List of questions that are not ‘Not answered’ or ‘Not adopted’) : In case there is no satisfactory and acceptable answer yet, questions which not completed will be collected and displayed together. Questioner will ‘Adopt’ the most helpful and satisfactory answer as the best answer among answers provided other users. Once an answer is adopted as the best answer by the questioner, the relevant question will be considered as a solved question. On the contrary, questions with no ‘Adopt’ will be considered as ‘Unsolved’ questions.
④ Category(Categories for search through classification) : Questions that are belonged to the relevant category can be separately searched.
⑤ Administration(Management function) : This is management menu provided only for Q&A administrator. Management menu consists of function to set Category classification system and policy management function to manage criteria for selection of popular Q&A and best answer.

3. Asking a Question

There is a built-in web editor for easy and convenient composition of question. In addition, input screen to compose question has many options such as notification setting to check answer right after posting of an answer, request for answer to expert in which questioner can designate a specific person as an answerer for convenience of question and answer activity.

The followings are description of each input items based on example screen for registration of question.

① Title : Enter a question title. Question title is a required item.
② Urgent question : In case of a question that needs an urgent answer, please check at urgent question item. Question checked as an urgent question can be searched by a list of urgent questions, and in case of recent question, it will be displayed in main screen for easier view of many users.
③ Category selection : The drop-down box next to urgent question check item is to select category of question. In case there is no category is related to a question, you may select 'no classified and miscellaneous'.
④ Answer registration alarm : This function is to inform questioner of the registration of answer. This alarm function allows questioner to quickly check out answer right after registration of answer. Media of mail, SMS, and/or message are available as a notification media and they can be used separately and in combination.
⑤ Content of question : Enter the detail of a question. Since there is a built-in web editor, it is easy and convenient to compose question contents.
⑥ File attachment : In case there is file related to question, you can select and attach it.
⑦ Tagging : Enter keywords related to a question. Please correctly enter related keyword for better application and sharing of a question. Max 10 keyword can be entered, and use coma (,) as a separator between keywords. You can change the keyword you entered any time.

4. Giving an Answer

If you find a question to which you can give an answer, please click 'Answer' button on the right bottom to move to answer screen in order to input answer. Not only expert requested for answer but also anybody can answer a question.

Read a question

Give an answer to the question

You can write answer to a question in answer screen. Answer to a question can be also composed in various formats using the built-in web editor. In case there is reference file related to answer, you can attach file as you do for question registration. For your information, tag value same with that of question screen will be set as default for tag for answer. Answerer can change it if necessary.

Answers by experts nominated as an answerer

Answer by the user designated as an answerer will be same with that of general answer. However, user designated as an answerer will receive the fact of question registration via media such as mail, SMS, or/and message. Accordingly, user designated as an answerer can answer to the designated question right away. The following message screen is to inform user designated as an answerer of registration of a question to which he/she is designated as an answerer. The user may click the question link included in message contents to check for question detail and to give an answer.

※ The above message linkage function is available when message module of this solution is established.

5. Choosing & Sharing

A questioner can select the most useful and helpful answer among answers. Answer selected by a questioner will be considered as the most appropriate problem solving method to the question and it will be marked with ‘Adopted’ answer for reference of other users. Only questioner can ‘Adopt’ the most useful and helpful answer among many answers.

Choose Asker's Best Answer

Person who asked the question will move to the screen to search the most helpful answer and click 'Choose as Best Answer' button on the bottom of search screen to select the most appropriate and helpful answer. Once a questioner selects an answer, the relevant question will be considered as to be completed.

Identify Asker's Best Answer

Chosen answers by askers will be marked with 'Chosen'. 'Chosen' mark is to show the most useful and helpful answer among many answers and for sharing of the information with other users. The questioner can cancel answer adoption by clicking 'Uncheck adoption' button.

Choose Reader's Best Answers

Other users can click ‘Recommend’ button to recommend the answer as reader's best answer. In case of getting recommendation above certain standard, Answer having recommendation counts more than certain criteria by will be marked as the 'Best' answer recommended by other many users, even though it is not the best answer selected by questioner. The number of users' recommendation to be a popular answer will be set according to 'Best answer management policy' decided by administrator.
