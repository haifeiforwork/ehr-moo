Checklist

순번 구분 상세 체크리스트 검수
Y N N/A
1 인식의 용이성 img, input 이미지 태그, 이미지맵 태그 옆에 alt=""를 사용하였으며 alt="" 내용은 동일한 표준에 의해 입력되고 있는가?
2 프로세스나 설명 위주의 이미지의 경우 동일하게 인식할 수 있는 대체텍스트를 소스 내에 작성하였는가?
3 동영상, 플래시 삽입시 대체텍스트를 제공하고 있는가?
또한 스캔 형식의 PDF 문서 등을 보여줄 때 스크린 리더기가 읽을 수 있는 실제 텍스트나 접근 가능한 html을 제공하고 있는가?
4 멀티미디어 콘텐츠에는 자막, 원고 또는 수화를 제공하고 있는가?
5 아이콘이나 이미지 등의 색상을 배제하고도 명확한 콘텐츠 전달, 구분이 되는가?
(그래프나 단색아이콘 등을 주로 체크)
6 텍스트 콘텐츠와 배경 간의 명도 대비가 12px normal 기준으로 4.5대 1 이상인가?
7 배경음은 자동으로 재생되지 않고 사용자의 선택에 의해 제공되고 있는가?
8 운용의 용이성 키보드만으로도 웹 콘텐츠가 제공하는 모든 기능을 수행할 수 있는가?
(특히 onmouse 관련 이벤트, 레이어, select box의 option 접근성 여부 점검)
9 키보드에 의한 초점은 논리적으로 이동하고 있으며, 시각적으로 구별할 수 있는가?
10 시간 제한이 있는 콘텐츠나 자동으로 변경되는 콘텐츠는 별도의 제어기능을 제공했는가?
11 깜빡이거나 번쩍이는 콘텐츠에 대한 경고여부가 있는가?
12 모든 콘텐츠 및 긴 목록형 화면 코딩시 반복 네비게이션 링크(skip navigation) 를 제공하고 있는가?
13 콘텐츠 그룹별로 <h1>~<h6> 태그를 일관성 있게 작성하였는가?
14 iframe 사용시 해당 내용을 이해할 수 있도록 적절한 제목(title 속성)을 제공하고 있는가?
15 브라우저 타이틀은 표준을 준수하고 있는가?
16 이해의 용이성 사용자가 의도하지 않은 기능(자동으로 팝업이 뜨거나 초점 변화가 일어나는 기능 등)은 없는가?
17 탭 키를 이용하여 양식 제어 요소 간을 이동할 때 좌상단에서 우하단으로 순차적인 이동이 가능하며 그 흐름이 논리적인가?
18 테이블의 summary, caption, thead, tbody, th, tr, td 등을 표준에 맞게, 일관성 있게 작성하였는가?
19 th scope 정보는 명확히 작성하였는가?
20 입력 서식에는 label과 id 또는 title을 표준에 맞게 입력하였는가?
(inputbox, textarea, checkbox, radio button 각각 표기법 확인)
21 견고성 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없는가?
22 애플릿, 플러그인(ActiveX, 실버라이트, 플래시 등) 등 부가 애플리케이션은 자체적인 접근성이나 대체 콘텐츠를 제공하고 있는가?
23 자바스크립트가 작동하지 않는 경우에 대비하여 접근시 문제가 없도록 콘텐츠가 제작되어 있는가?
